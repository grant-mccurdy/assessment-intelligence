# SQL Extract Data Dictionary

This document describes the public-safe SQL extract contract consumed by
`assessment-intelligence`.

All rows are synthetic demo data generated by the sibling
`synthetic-education-data` project. These extracts must not be replaced with
real Canvas exports, real rosters, private gradebooks, student contact data, or
school-private records in this public repository.

## Sources

The same extract family can be built from two sources:

- Local DuckDB warehouse: `../synthetic-education-data/warehouse/synthetic_math.duckdb`
- Hosted Supabase public views: `synthetic-education-data` public Data API views

DuckDB remains the reproducible local baseline. Supabase demonstrates a hosted
serving layer with a narrow public view contract. The base `lms` and
`analytics` schemas are not part of the public API contract.

## Extract Files

| File | Row Grain | Primary Use |
| --- | --- | --- |
| `course_section_performance.csv` | course section by populated assignment | Section-level score and participation reporting |
| `assignment_growth_by_course.csv` | grade and course | Assignment 01 to Assignment 02 growth diagnostics |
| `nonparticipation_by_group.csv` | assignment, grade, attendance category, course track | Missingness and participation checks |
| `lms_enrollment_reconciliation.csv` | course section and reconciliation status | LMS roster quality reporting |
| `student_readiness_extract.csv` | synthetic student enrollment record | Readiness and student-level synthetic dashboard fixtures |

## `course_section_performance.csv`

One row per course section and populated assignment window.

| Column | Meaning |
| --- | --- |
| `course_id` | Synthetic course identifier. |
| `course_name` | Public-safe course label. |
| `course_track` | Course track such as regular, honors, AP, or beyond-core. |
| `section_id` | Synthetic section identifier. |
| `section_label` | Public-safe section label. |
| `teacher_id` | Synthetic teacher identifier. |
| `teacher_label` | Public-safe teacher label. |
| `assignment_label` | Generic assignment label. |
| `sequence_index` | Assignment order in the assessment sequence. |
| `assessment_window` | Assessment timing label. |
| `enrolled_students` | Synthetic enrolled student count. |
| `present_students` | Count with present score evidence. |
| `nonparticipation_rate` | Share of enrolled rows without present score evidence. |
| `avg_present_score` | Average score among present students only. |
| `min_present_score` | Minimum present-student score. |
| `max_present_score` | Maximum present-student score. |

## `assignment_growth_by_course.csv`

One row per grade and course where both Assignment 01 and Assignment 02 have
present-student score evidence.

| Column | Meaning |
| --- | --- |
| `grade_level` | Synthetic student grade level. |
| `course_id` | Synthetic course identifier. |
| `course_name` | Public-safe course label. |
| `course_track` | Course track. |
| `matched_students` | Count with matched Assignment 01 and Assignment 02 scores. |
| `assignment_01_avg` | Assignment 01 average among matched students. |
| `assignment_02_avg` | Assignment 02 average among matched students. |
| `avg_observed_growth_delta` | Average Assignment 02 minus Assignment 01 score. |
| `min_observed_growth_delta` | Minimum observed growth delta. |
| `max_observed_growth_delta` | Maximum observed growth delta. |

## Public Safety Rules

- Keep raw LMS-style rows, real rosters, contact data, and private gradebooks out of this repo.
- Treat OpenAI review output as advisory documentation only.
- Use `SYNTHETIC_EDUCATION_SUPABASE_*` env aliases for hosted extraction so generic Supabase keys cannot point this repo at the wrong project.
- Re-run `make privacy-check`, `make sql-warehouse-report`, and `make supabase-report` after changing extract logic.
