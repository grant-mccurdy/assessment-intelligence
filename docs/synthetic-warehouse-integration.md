# Synthetic Warehouse Integration

`assessment-intelligence` can query the DuckDB warehouse generated by the sibling `education-data-simulation-engine` project. It can also run a hosted Supabase extraction against selected public read-only views from the validated synthetic warehouse.

The intended portfolio workflow is:

```text
education-data-simulation-engine
-> synthetic Canvas-like artifacts
-> DuckDB raw/raw_canvas schemas
-> star-schema marts
-> hosted Supabase public views
-> assessment-intelligence SQL extracts
-> analysis, reports, and dashboards
```

## Build Source Warehouse

From `education-data-simulation-engine`:

```bash
make analytics-install
make warehouse
```

This creates:

```text
../education-data-simulation-engine/warehouse/synthetic_math.duckdb
```

## Run SQL Extracts

From `assessment-intelligence`:

```bash
make analytics-install
make synthetic-warehouse-extract
```

The extract target runs SQL files from:

```text
sql/extracts/
```

and writes public-safe CSV outputs to:

```text
data/external/education-data-simulation-engine/
```

It also writes a summary report:

```text
reports/sql_warehouse_assessment_extract.md
```

For a more analyst-facing brief, run:

```bash
make sql-warehouse-report
```

This writes:

```text
reports/sql_warehouse_assessment_report.md
```

## Data Boundary

The DuckDB source warehouse and all extracts are synthetic. Do not point this workflow at real Canvas data, raw institutional exports, or private gradebooks in the public repository.

## Hosted Supabase Query Mode

DuckDB remains the reproducible local source for public reviewers. Supabase is an optional hosted serving layer for remote extraction from the populated synthetic warehouse.

The hosted extractor uses the Supabase Data API by default. It reads selected public views from the `education-data-simulation-engine` project using local credentials from the shared local env file. Database URLs, access tokens, and publishable keys must stay out of this repo.

Expected local aliases are:

- `SYNTHETIC_EDUCATION_SUPABASE_URL`
- `SYNTHETIC_EDUCATION_SUPABASE_PUBLISHABLE_KEY`

From `assessment-intelligence`, run:

```bash
make analytics-install
make supabase-extract
make supabase-report
```

The hosted extract writes public-safe CSV outputs to:

```text
data/external/education-data-simulation-engine-supabase/
```

and writes summary reports to:

```text
reports/supabase_assessment_extract.md
reports/supabase_assessment_report.md
```

Current hosted public views include:

- `course_section_performance`
- `assignment_growth_by_course`
- `nonparticipation_by_group`
- `lms_enrollment_reconciliation`
- `student_readiness_extract`
- `warehouse_summary`

The base `lms` and `analytics` schemas remain outside the public API contract. Use the Management API read-only SQL mode only as a local debugging fallback, not as the normal consumer path.
